How to use the profileAAR-Plugin

This tool is created to transform fotogrammetric control points from archaeological excavation, to use them to georeference profiles is Qgis.

1. Input data

At first we need the dataset.

All we need is a shapefile with the fotogrammetric control points.

As attributes we need:

  • the z-value [float/double/real]
  • the label of the profile (e.g. number of the profile [text or number])
  • the view direction (defined by the position of the observer related to the profile as N, E, S or W [text])
  • the used Points (points that are used to determine the slope of the regression [0/1])

    • If a profile is NE - SW orientated it doesn't matter if you choose E or S.

All profiles and points will be checked for errors that can cause problems while rectifing the images or transforming the points.

  • all profiles need at least 4 points
  • one profile can have only one direction of view
  • the view directions have to be one of the four cardinal directions
  • the layer has to have an projected CRS
  • the used Points column contains only 0 or 1

    • You will get the best results, if your are careful on the excavation and prepare a perfectly straight profile!

2. You have the choice

At first you have to select the layer, then the attributes are visible in the z coord, the pr number, the use and the view column boxes.

In this case use means if the point will be used to calculate the rotation angle (cloumn with '1'). If there is an outlier in the data, mark it with a '0'.

Please select the matching attributes.

In the next two boxes, you have to choose:

    In the Method box:

  • 1.: Projected. The profile will be projected on a vertical projection pane. This is like a drawing in the field.
  • 2.: Surface. The profile will be projected on a pane, that is parallel to the profile. This is like drawing the surface of the profile.

    • View on a profile (0). The result after choosing projected (1) or surface (2).

    In the Direction box:

  • 1.: Original. The points will be parallel to the profile.
  • 2.: Horizontal. The points will be parallel to the x-axis.

    • View on a profile(0). The result after choosing original (1) or horizontal (2).

Export elevation points will export an additional shapefile containing the upper right fotogrammetric control point and its height and the profile number.

This can be very helpful while processing the drawings.

3. Run the calculation

If you press the OK Button the magic happens.

Some hints for a good result

  • If you have a straight profile you will have the best results
  • If profiles are very long and not very straight, sometimes it is better to split them (we plan a function for the next version to do it the easy way for the user)

4. Accuracy

Getting values for the accuracy of the points.

When you open the attribute table of the rotated points, you will find the column distance.

This value represents the distance of each point to the calculated profile between the points. As higher the value is, as worse is the result.

If there is one point not in the line, this one will have a high distance. If you choose 0 in the original column used Points, this point will be excluded

in the calculation of the regression line, but the point will be rotated. This can be used to correct measurement faults.
